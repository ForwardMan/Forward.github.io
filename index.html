分治法求最近点对

发表于 2016-06-27

扩展欧几里得算法

已经知道gcd可以求两个数的最大公约数。
已知一定存在 xa + xb = gcd(a,b)（x,y皆为整数） , 求解未知数x,y.

模板

int extend_gcd(int a,int b,int &x,int &y)
{
if(b == 0)//当b = 0;则此时根据xa+yb = gcd(a,b) 则gcd(a,b) = a,x = 1, y = 0;
{
x = 1;
y = 0;
return a;//返回最大公倍数
}
int ans = extend_gcd(b,a%b,x,y);//递归直到b = 0.
int t = x;
x = y;
y = t - a/b*y;
return ans;
}

原理

当实现一次递归时，a’ = b, b’ = a%b; 且有xa’ + yb’ = gcd(a’,b’);
因为 b’ = a%b = a - a/bb; gcd(a’,b’) = gcd(a,b);
所以 xb + y(a-a/bb) = gcd(a,b);
整理得 ya + b(x-a/by) = gcd(a,b);
所以 x = y, y = a-a/by; 就可以求出x,y.

分治法求最近点对

发表于 2016-05-16

问题

给出一大堆点p1(x1,y1),p2(x2,y2)….pn(xn,yn),求xoy平面中距离最近的两个点。

方法

思路

分治法，先按x升序排列，用递归求除了在中点左边和中点右边的最小两点间距离d_min，然后再选择x坐标为(mid-d_min,mid+d_min)之间的距离，此时按照y升序排列，然后在求出其最小值（这里不能用递归了，直接比较）。

代码板子

struct point//用结构体保存每一个点
{
double x,y;
}p[maxn];

int a[maxn];//保存筛选的坐标点的索引即2min(dl,dh)区间的坐标点索引

int cmpx(point a,point b)//对n个点按横坐标由小到大排序
{
return a.x<b.x;
}

int cmpy(int a,int b)//（这里用的是下标索引）对筛选的点按纵坐标由小到大排序
{
return p[a].y<p[b].y;
}

inline double dis(point a,point b)//求点对间的距离
{
return sqrt((a.x-b.x)(a.x-b.x)+(a.y-b.y)(a.y-b.y));
}

double closest(int low,int high)//求最近点对
{
int i,j,k;
if(low+1==high)//只有两个点
return dis(p[low],p[high]);
if(low+2==high)//只有三个点
return min(dis(p[low],p[high]),min(dis(p[low],p[low+1]),dis(p[low+1],p[high])));
int mid=(low+high)>>1;//求中点即左右子集的分界线
double d=min(closest(low,mid),closest(mid+1,high));//此处递归，递归出除了在中点左边和中点右边的最小两点间距离d_min
for(i=low,k=0;i<=high;i++)//找出x坐标为(mid-d_min,mid+d_min)的点
{
if(p[i].x >=p[mid].x-d&&p[i].x<=p[mid].x+d)
a[k++]=i;//保存这些点的下标索引
}
sort(a,a+k,cmpy);//按y坐标进行升序排序
for(i=0;i=d)//如果两点间y的差值都比d_min大了，那么该两点的距离肯定比d_min大，所以这样的点直接break，不考虑。
break;
d=min(d,dis(p[a[i]],p[a[j]]));
}
}
return d;
}
int main()
{
int i,n;
while(scanf(“%d”,&n) && n!= 0)//n个点
{
for(i = 0 ; i < n ; ++i)
scanf(“%lf %lf”,&p[i].x,&p[i].y);
sort(p , p + n , cmpx);//按x坐标进行升序排序
printf(“%.2f\n”,closest(0 , n - 1)/2);//最近点对间的距离
}
return 0;
}

大学读的第一本书 ———— 《城南旧事》

发表于 2016-05-12

介绍

《城南旧事》作者，现代盛名女作家林海音。主要内容是以童年英子的眼睛，叙述在北京城南街道的生活往事。林海音，原籍台湾，生于日本，长于北京。这本书堪称她独闯文坛数十年的经典之作。书中文字与水彩图图交替出现，形象生动，触动人心。绘图者，关维兴，现代国际水彩画大牌一名。

偶然

去年刚上大学不久后，就迎来了国庆。国庆期间，去了天安门广场看升旗，也去了王府井大街吃早餐、还去了国家图书馆。那时只去了国家典籍图书馆，那时傻傻的、还不知道旁边有个巨大的建筑物是现代图书馆。所以那时在典籍图书馆也逛得够无聊的、最后走的时候看见个国图小书店，和我一同去的小舒就跩我进去买本书当做纪念，当时并不想买什么书，就当做陪陪他也就进去了。溜达了一会，也漫不经心的，看什么书都不上眼、最后在一个角落看见了一个书名立即就吸引了我 ———— 《城南旧事》 这个书名好，看到就有一种说不出的感觉， 也许是我天生的恋旧情怀、一种潜在的执念就让我掏出了钱包，买了走人。

读书

这本书买回来后也一直没看过，放在床头保留很久、可能是忙也可能是没有看书的念头。前几天、听stay说起读书的益处，能驱走学生的烦恼与迷茫、加上我最近就有点迷茫、又加上整天在值班无聊，昨天在整理床铺时就顺便带上这本书去了值班处、一看就停不下来了，就这样我用了两天时间把这本书看完了。

品书

书中文字简洁，没有读不懂的难字。一幅幅水彩画让我感到平凡感情的魅力。童年，一条巷子、几次搬迁，惠安馆的疯子、常被抽打的妞儿、草丛中的小贼、跟德先叔走的兰姨娘、失去孩子的宋妈、病逝的父亲… 小学念的：一去二三里、烟村三四家、亭台六七座、八九十枝话；小学毕业歌：长亭外，古道边，芳草碧连天… 一个个元素在童年的缩影中浮现、永远也解不开的疑问、脑海里永远搞不明白的事与人，还有长大成人肩负的责任。

关于近期转专业浪潮引起的对未来的遥想。

发表于 2016-05-09

电子是否有未来

这段时间全校聚焦转专业的问题，我们电子的就有转出3人，Stay、cho·ose两位大神和挚友都将远离电子投奔计算机、这让我一下感到被鼓动、再加上这次的双选会、听说电子的无一家公司招聘，连亲爱的瑞姐都劝我们要不要转专业、说她被面试的时候一脸懵逼。。。人家都不要你这种专业的。很多人都对是否留在电子产生了疑问，包括我。我们都在疑问这个专业是否已被社会淘汰、这个专业是否早已没有了未来。。。

当初 。

漫步在校园的主道上、我脑子里想着这件事。我回想起了当时填志愿的时候，我记得我把在网吧呆了好多天，就是为了了解各种专业，希望能找到自己钟情的专业。我记得当时我看了很多，都觉得很乏味，为什么我对这些专业都那么提不起劲来~？ 再经过一个星期的翻找翻找、我终于对两个专业产生了兴趣，一个是临床医学、另一个就是电子。当时我也有问我叔叔（我叔叔对我影响挺大，我几乎什么事情都是听他的），他也思考了很久，最后他给我的回答是：凡是一切管理类的、行政类的、电子类的都不要选！我听了后一脸慌张、行政和管理不要我可以接受，可电子，为什么啊。没多问，后来我说那我选学医吧，然后在看了各大医校的分数线、屁啊~我哪有那么高的分数啊！ 心理切想着，还是喜欢电子。当初叔叔不在家，也没有经常给我施加压力、他就说，除了以上那些不要选的，其他的任由我选，说填好了志愿就和他说了一声是什么专业。我也没多说，就一个字，嗯。后来叔叔知道后，也没说什么，还是一脸高兴的为我主办了一场升学酒宴。那时，我就在心理默念：对不起，叔，我一定会出人头地的 。

不变初心、用一个兴趣加真心、拼下一个未来

现在是什么情况、不就是两位大神朋友转向了计算机而已嘛、不就是努力读书的学姐找不到工作而已嘛。我凭什么就对我自己的未来产生疑问了。我的初心还是没有变，我还是热爱电子这个专业、只是我还太慵懒、学到的知识太少、我的所有担忧其实就是我自己没有足够认真学习所产生的意象而已，如果我真转了专业、我才会后悔一辈子。未来茫茫、其实就在一念之间。未来也并不是遥遥无期、只要我坚持着、用心着、也一定能打下一片事业。转专业、三个字、怎么写？

学上一身本领、他日再来讨教 ！

【数论】gcd && lcm

发表于 2016-05-02

gcd（欧几里德）求最大公约数

用一般的方法去求最大公约数会费时很多，如果位数上升到18位数的时候，再一个个的去求，那就太浪费生命了。
那就用一个高效的gcd来解吧，代码很简单，如下：
typedef long long ll;
ll gcd(ll a,ll b)
{
if(b == 0)
return a;
return gcd(b,a%b);
}

这代码通俗易懂，很简短，却也很高能，把它牢记于心~

gcd 和 lcm 联合起来求 最小公倍数

其实这个就是用a乘b的值在除以gcd就行了，代码：

typedef long long ll;
ll gcd(ll a,ll b)
{
if(b == 0)
return a;
return gcd(b,a%b);
}
ll lcm(ll a,ll b)
{
return a*b/gcd(a,b);
}
